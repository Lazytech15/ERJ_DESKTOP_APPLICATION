# ERJ Smart Solution – Attendance Management System

React (Vite) frontend + Electron backend.

## Structure
```
erj-attendance/
├── electron/            # Electron main process (backend)
│   ├── main.js           # Window, IPC handlers, attendance data
│   ├── preload.js         # Secure IPC bridge (window.electronAPI)
│   └── dist/              # Built frontend (generated by `npm run build:frontend`)
├── frontend/             # React + Vite app (UI)
│   ├── index.html
│   ├── vite.config.js
│   └── src/
│       ├── App.jsx
│       ├── main.jsx
│       ├── components/   # Header, Scanner, StatsRow, EmployeeList, DetailCard, ClockOutCard, Titlebar, Icons
│       ├── hooks/         # useClock
│       ├── services/      # api.js (window.electronAPI wrapper)
│       ├── data/          # sample/mock data for browser dev mode
│       ├── styles/        # CSS (per-component)
│       └── utils/
├── assets/
│   └── icon.svg
└── package.json          # root scripts orchestrate frontend + electron
```

## Setup
```bash
npm install
```
This also installs the `frontend/` dependencies (via `postinstall`).

## Development
Runs the Vite dev server and Electron together (hot reload):
```bash
npm run dev
```

## Production build / run
Builds the React app into `electron/dist` and launches Electron against it:
```bash
npm start
```

## Package the desktop app
```bash
npm run dist
```

## IPC API (window.electronAPI)
Exposed by `electron/preload.js`:
- `getAttendanceState()` – fetch current employees, clock-outs, stats
- `scanEmployee({ value, mode })` – lookup employee by ID or name
- `clockAction({ employeeId })` – toggle clock in/out for an employee
- `refreshAttendance()` – re-fetch attendance data
- `onAttendanceUpdate(callback)` – subscribe to live state pushes
- `openSettings()` / `openEmployeeDirectory()` – stubs for future modals
- `minimizeWindow()` / `maximizeWindow()` / `closeWindow()` / `getVersion()`

Replace the in-memory demo data in `electron/main.js` with real database/API logic.
